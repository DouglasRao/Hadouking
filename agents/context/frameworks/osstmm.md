# OSSTMM Reference (Operational Security Testing)

## Purpose
Use OSSTMM principles for disciplined, measurable security testing across people, process, and technology.

## Core Principles
- Scope precision over broad uncontrolled testing.
- Measure attack surface and trust boundaries.
- Track repeatable test evidence and risk metrics.
- Separate observation from interpretation.

## Channel-Oriented Thinking
- Human channel (social vectors)
- Physical channel (onsite exposure)
- Wireless channel (RF / Wi-Fi exposure)
- Telecommunications channel
- Data networks channel

## Practical Pentest Alignment
- Establish rules of engagement before execution.
- Test controls as implemented, not as documented.
- Validate detection and response, not only prevention.
- Include operational impact and recovery assumptions.

## Execution Guidance
- Prefer low-noise, high-signal actions.
- Maintain strict evidence logging and timeline integrity.
- Stop any action that risks violating safety constraints.
